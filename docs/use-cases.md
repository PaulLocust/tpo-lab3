# Прецеденты использования — T-Путешествия (tbank.ru/travel)

В работе тестируется только UI-функциональность: фильтры, отображение
информации в карточках, переходы между разделами и на партнёрские сайты.
Формы, требующие ввода персональных данных (e-mail, номер телефона,
оплата), **не тестируются**, так как сайт работает на проде.

---

## UC-1. Выбор авиабилетов

| Поле                  | Значение                                                                              |
|-----------------------|---------------------------------------------------------------------------------------|
| **ID**                | UC-1                                                                                  |
| **Краткое описание**  | Найти подходящий авиабилет по маршруту и применить фильтры выдачи                     |
| **Главный актёр**     | Гость                                                                                 |
| **Предусловия**       | Открыта страница `/travel/flights/`                                                   |
| **Постусловия**       | Отображён список рейсов, удовлетворяющих фильтрам                                     |
| **Связанные UC**      | `<<include>>` UC-7, UC-8; `<<extend>>` ← UC-9, UC-10, UC-11, UC-12, UC-13; ассоциация — UC-21 |

**Основной поток:**

1. Гость указывает маршрут «Откуда → Куда» (`<<include>>` UC-7).
2. Гость указывает дату вылета (`<<include>>` UC-8).
3. Гость нажимает кнопку «Найти».
4. Система открывает страницу выдачи рейсов.
5. Гость опционально применяет фильтры (`<<extend>>` UC-9 – UC-13).
6. Система обновляет список рейсов в соответствии с выбранными фильтрами.

**Альтернативный поток:** не указан маршрут или дата — поиск не запускается,
система остаётся на форме поиска.

## UC-2. Выбор отеля

| Поле                  | Значение                                                                              |
|-----------------------|---------------------------------------------------------------------------------------|
| **ID**                | UC-2                                                                                  |
| **Краткое описание**  | Найти отель в указанном городе, применить фильтры, открыть карточку отеля             |
| **Главный актёр**     | Гость                                                                                 |
| **Предусловия**       | Открыта страница `/travel/hotels/`                                                    |
| **Постусловия**       | Отображён список отелей либо открыта карточка выбранного отеля                        |
| **Связанные UC**      | `<<include>>` UC-14, UC-8; `<<extend>>` ← UC-15, UC-16, UC-17, UC-18; ассоциация — UC-22 |

**Основной поток:**

1. Гость указывает город назначения (`<<include>>` UC-14).
2. Гость указывает даты заезда / выезда (`<<include>>` UC-8).
3. Гость нажимает «Искать».
4. Система открывает страницу выдачи отелей.
5. Гость опционально применяет фильтры (`<<extend>>` UC-15, UC-16, UC-17).
6. Гость опционально добавляет отель в избранное (`<<extend>>` UC-18).
7. Гость кликает по карточке — система открывает страницу отеля.

**Альтернативный поток:** не указан город — поиск не запускается.

## UC-3. Выбор билетов на поезд

| Поле                  | Значение                                                                              |
|-----------------------|---------------------------------------------------------------------------------------|
| **ID**                | UC-3                                                                                  |
| **Краткое описание**  | Найти ЖД-билет через партнёрский сервис (OneTwoTrip)                                  |
| **Главный актёр**     | Гость                                                                                 |
| **Второстепенный**    | Внешний партнёрский сайт                                                              |
| **Предусловия**       | Открыта страница `/travel/trains/`                                                    |
| **Постусловия**       | Открыта вкладка партнёрского сайта                                                    |
| **Связанные UC**      | `<<include>>` UC-19                                                                    |

**Основной поток:**

1. Гость заходит в раздел «Поезда».
2. Гость нажимает кнопку поиска (`<<include>>` UC-19).
3. Система выполняет редирект (или открывает новую вкладку) на партнёрский
   сайт OneTwoTrip с параметрами запроса.

**Замечание:** так как ЖД-поиск полностью делегирован партнёру, в рамках
лабораторной работы тестируется только корректность перехода на партнёрский
домен.

# Вспомогательные прецеденты (include / extend)

## UC-7. Указать маршрут «Откуда → Куда» (`<<include>>` UC-1)

1. Гость кликает по полю «Откуда» и вводит название города.
2. Система отображает выпадающий список подсказок.
3. Гость выбирает город из списка — поле заполняется.
4. Гость аналогично заполняет поле «Куда».

## UC-8. Указать дату (`<<include>>` UC-1, UC-2)

1. Гость кликает по полю даты.
2. Система открывает виджет календаря.
3. Гость выбирает ячейку нужной даты.

## UC-9. Применить фильтр «Без пересадок» (`<<extend>>` UC-1)

1. Гость в фильтрах выдачи рейсов активирует «Без пересадок».
2. Система оставляет в выдаче только прямые рейсы.
3. **Проверка:** в карточке каждого рейса присутствует только один сегмент
   (две точки маршрута: пункт отправления и пункт назначения).

## UC-10. Сортировка по цене (`<<extend>>` UC-1)

1. Гость выбирает сортировку «Сначала дешевле».
2. Система сортирует выдачу по возрастанию цены.
3. **Проверка:** цена каждой следующей карточки `>=` цены предыдущей.

## UC-11. Фильтр авиакомпании (`<<extend>>` UC-1)

1. Гость в фильтрах выбирает конкретную авиакомпанию (чекбокс).
2. Система оставляет только рейсы выбранной компании.
3. **Проверка:** в карточке каждого рейса фигурирует выбранная авиакомпания.

## UC-12. Фильтр «С багажом» (`<<extend>>` UC-1)

1. Гость включает фильтр «С багажом».
2. Система оставляет только рейсы, в стоимость которых включён багаж.
3. **Проверка:** в карточках появляется индикатор включённого багажа.

## UC-13. Фильтр времени вылета (`<<extend>>` UC-1)

1. Гость задаёт интервал времени вылета.
2. Система обновляет выдачу с учётом интервала.
3. **Проверка:** время вылета в карточке попадает в заданный диапазон.

## UC-14. Указать город назначения (`<<include>>` UC-2)

1. Гость кликает по полю поиска отелей и вводит город.
2. Система отображает выпадающий список с подсказками.
3. Гость выбирает вариант.

## UC-15. Сортировка отелей по цене (`<<extend>>` UC-2)

1. Гость выбирает «Сначала дешевле».
2. Система сортирует выдачу по возрастанию цены.
3. **Проверка:** цена каждой следующей карточки `>=` цены предыдущей.

## UC-16. Фильтр «Спецпредложения» (`<<extend>>` UC-2)

1. Гость в фильтрах активирует «Спецпредложения».
2. Система оставляет только отели со скидкой.
3. **Проверка:** в карточке отображается процент скидки или индикатор
   спецпредложения.

## UC-17. Фильтр «Бесплатная отмена» (`<<extend>>` UC-2)

1. Гость активирует фильтр «Бесплатная отмена».
2. Система оставляет только отели с этой опцией.
3. **Проверка:** в карточке есть отметка «Бесплатная отмена».

## UC-18. Добавление отеля в избранное (`<<extend>>` UC-2)

1. Гость в карточке отеля нажимает иконку «сердечко».
2. Система помечает отель как избранное.
3. Гость переходит в раздел «Избранное».
4. **Проверка:** выбранный отель присутствует в списке избранного.

## UC-19. Переход на партнёрский сайт (`<<include>>` UC-3, UC-5, UC-6)

1. Гость нажимает кнопку «Найти на …» / «Подобрать» / «Выбрать».
2. Система выполняет редирект или открывает новую вкладку.
3. **Проверка:** домен новой страницы — это партнёрский домен (не `tbank.ru`).

## UC-20. Открытие карточки тура у партнёра (`<<include>>` UC-4)

1. Гость нажимает «Найти тур» на карточке партнёра в `/travel/tours/`.
2. Система открывает страницу тура у партнёра.
3. **Проверка:** новая вкладка/страница ведёт на корректный URL партнёра.

## UC-21. Переход на покупку авиабилета (ассоциация с UC-1)

1. На странице выдачи рейсов гость нажимает «Выбрать» на интересующей карточке.
2. Система раскрывает детализацию рейса с кнопкой «Продолжить».
3. Гость нажимает «Продолжить».
4. Система выполняет переход на страницу оформления покупки
   `https://www.tbank.ru/travel/flights/checkout/`.
5. **Проверка:** URL содержит `/travel/flights/checkout`.

**Замечание:** в рамках UI-теста на проде взаимодействие со страницей
checkout ограничено проверкой факта перехода — никакие персональные данные
не вводятся.

## UC-22. Переход к бронированию отеля (ассоциация с UC-2)

1. На странице выдачи отелей гость открывает карточку выбранного отеля.
2. На странице отеля гость нажимает «Выбрать номер».
3. Система выполняет переход на страницу оформления бронирования
   `https://www.tbank.ru/travel/hotels/new/checkout`.
4. **Проверка:** URL содержит `/travel/hotels/new/checkout`.

**Замечание:** на странице checkout также никакие персональные данные не вводятся.
