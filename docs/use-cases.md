# Прецеденты использования — T-Путешествия (tbank.ru/travel)

---

| **Прецедент: Просмотр главной страницы T-Путешествий** |
|---|
| **ID:** 1 |
| **Краткое описание:** Открыть главный раздел путешествий и убедиться, что доступны основные подразделы |
| **Главный актер:** Гость |
| **Второстепенные актеры:** нет |
| **Предусловия:** нет |

**Основной поток:**
1. Пользователь переходит на `https://www.tbank.ru/travel/`.
2. Система отображает главную страницу с логотипом T-Банка.
3. Система отображает навигацию по подразделам: Авиа, Отели, Поезда, Туры.
4. Пользователь визуально оценивает доступный функционал.

---

| **Прецедент: Переход между разделами сайта** |
|---|
| **ID:** 2 |
| **Краткое описание:** Перейти из главного раздела в нужный подраздел (авиабилеты, отели, ЖД-билеты, туры) |
| **Главный актер:** Гость |
| **Второстепенные актеры:** нет |
| **Предусловия:** Пользователь находится на любой странице раздела `/travel/` |

**Основной поток:**
1. Пользователь нажимает в шапке на ссылку нужного подраздела.
2. Система выполняет переход на соответствующий URL (`/travel/flights/`, `/travel/hotels/new/`, `/travel/trains/`, `/travel/tours/`).
3. Система отображает форму поиска выбранного подраздела (для подразделов с формой).

---

| **Прецедент: Поиск авиабилетов** |
|---|
| **ID:** 3 |
| **Краткое описание:** Найти подходящие рейсы по маршруту и дате |
| **Главный актер:** Гость |
| **Второстепенные актеры:** нет |
| **Предусловия:** Открыта страница `/travel/flights/`; поле «Откуда» предзаполнено городом по умолчанию (Санкт-Петербург) |

**Основной поток:**
1. Пользователь при необходимости меняет город отправления в поле «Откуда».
2. Пользователь вводит город назначения в поле «Куда» (`include` UC-10).
3. Система отображает выпадающий список подсказок городов.
4. Пользователь выбирает город из подсказок.
5. При необходимости пользователь выбирает дату вылета («extend» UC-08).
6. При необходимости пользователь открывает селектор пассажиров и класса («extend» UC-09).
7. Пользователь нажимает кнопку «Найти».
8. Система перенаправляет на страницу выдачи рейсов.

**Альтернативный поток (пустая форма):**

7а. Пользователь не заполнил «Куда»: система блокирует переход и не открывает выдачу.

---

| **Прецедент: Поиск отелей** |
|---|
| **ID:** 4 |
| **Краткое описание:** Найти отель в указанном городе на нужные даты |
| **Главный актер:** Гость |
| **Второстепенные актеры:** нет |
| **Предусловия:** Открыта страница `/travel/hotels/new/`; диапазон дат и число гостей предзаполнены значениями по умолчанию |

**Основной поток:**
1. Пользователь вводит название города в поле направления.
2. Система отображает подсказки городов / отелей (`include` UC-10).
3. Пользователь выбирает вариант из подсказок.
4. При необходимости пользователь меняет даты заезда/выезда и количество гостей.
5. Пользователь нажимает кнопку «Искать».
6. Система отображает страницу результатов с доступными отелями.

**Альтернативный поток (пустое направление):**

5а. Поле направления не заполнено — система не запускает поиск.

---

| **Прецедент: Поиск ЖД-билетов** |
|---|
| **ID:** 5 |
| **Краткое описание:** Найти ЖД-билеты по маршруту на нужную дату |
| **Главный актер:** Гость |
| **Второстепенные актеры:** партнёр-сервис OneTwoTrip (внешний редирект) |
| **Предусловия:** Открыта страница `/travel/trains/`; поле «Пассажиры» предзаполнено значением «1 взрослый» |

**Основной поток:**
1. Пользователь вводит город отправления в поле «Откуда» (`include` UC-10).
2. Пользователь вводит город назначения в поле «Куда».
3. При необходимости пользователь выбирает дату в поле «Когда».
4. Пользователь нажимает кнопку «Найти на OneTwoTrip».
5. Система открывает выдачу поездов (на партнёрском сайте).

**Альтернативный поток (пустая форма):**

4а. Незаполненные «Откуда» или «Куда» — система не запускает поиск.

---

| **Прецедент: Просмотр раздела «Туры»** |
|---|
| **ID:** 6 |
| **Краткое описание:** Просмотреть предложения от партнёров-турагентов |
| **Главный актер:** Гость |
| **Второстепенные актеры:** партнёры-турагенты (Travelata, Onlinetours, Level.Travel) |
| **Предусловия:** нет |

**Основной поток:**
1. Пользователь переходит на `/travel/tours/`.
2. Система отображает карточки партнёров-турагентов с кратким описанием.
3. У каждого партнёра есть кнопка-ссылка «Найти тур», ведущая на сайт партнёра.
4. Пользователь выбирает интересного партнёра и переходит к нему.

**Замечание:** в текущей версии у T-Travel нет собственной инлайн-формы поиска тура на этой странице — её роль выполняют партнёрские виджеты.

---

| **Прецедент: Расширенные опции авиапоиска (мульти-маршрут / командировка)** |
|---|
| **ID:** 7 |
| **Краткое описание:** Использовать дополнительные опции формы поиска авиа: «Сложный маршрут» и тоггл «Лечу по работе» |
| **Главный актер:** Гость |
| **Второстепенные актеры:** нет |
| **Предусловия:** Открыта форма авиабилетов («extend» UC-03) |

**Основной поток:**
1. Пользователь хочет либо составить мульти-сегментный маршрут, либо отметить, что летит по работе.
2. Для мульти-маршрута: нажимает «Сложный маршрут» → форма переключается в мульти-сегментный режим.
3. Для деловой поездки: переключает тоггл «Лечу по работе» → опция применяется к поиску.

**Замечание:** в актуальной версии формы T-Travel **отсутствует кнопка обмена городов местами (swap)**, поэтому исходно запланированный UC-07 «Swap» был заменён на эти реально доступные опции.

---

| **Прецедент: Выбор даты вылета через календарь** |
|---|
| **ID:** 8 |
| **Краткое описание:** Указать дату вылета с помощью виджета календаря |
| **Главный актер:** Гость |
| **Второстепенные актеры:** нет |
| **Предусловия:** Открыта форма авиабилетов («extend» UC-03) |

**Основной поток:**
1. Пользователь нажимает на поле «Когда».
2. Система открывает виджет календаря (ячейки `gridcell`).
3. Пользователь выбирает нужную дату в календаре.

---

| **Прецедент: Открытие селектора пассажиров и класса обслуживания** |
|---|
| **ID:** 9 |
| **Краткое описание:** Указать количество пассажиров и класс для авиабилета |
| **Главный актер:** Гость |
| **Второстепенные актеры:** нет |
| **Предусловия:** Открыта форма авиабилетов («extend» UC-03); по умолчанию выставлено «1 пассажир, эконом» |

**Основной поток:**
1. Пользователь нажимает на селектор пассажиров (поле «Пассажиры»).
2. Система отображает индикатор раскрытия `Arrow_opened` и открывает панель настроек.
3. Пользователь меняет число пассажиров или класс.
4. Система отражает изменения в селекторе и применяет их к поиску.

---

| **Прецедент: Автодополнение городов на любой форме** |
|---|
| **ID:** 10 |
| **Краткое описание:** Получать подсказки городов при вводе на любой форме поиска (авиа/отели/ЖД) |
| **Главный актер:** Гость |
| **Второстепенные актеры:** нет |
| **Предусловия:** Открыта одна из форм поиска (`include` UC-03 / UC-04 / UC-05) |

**Основной поток:**
1. Пользователь начинает ввод названия города в поле формы.
2. Система отправляет запрос на сервис подсказок.
3. Система отображает выпадающий список с подсказками
   (для авиа — `li[data-qa-type='itemColumn']`;
   для отелей — `div[role='option' и @data-qa-type='inputLocation.item']`;
   для ЖД — `div[role='option' и @data-qa-type='tui/dropdown-menu-item']`).
4. Пользователь выбирает один из вариантов (клавишами `↓` + `Enter` или кликом).
5. Поле заполняется выбранным значением.

**Альтернативный поток (бессмысленный ввод):**

3а. Введённая строка не соответствует ни одному городу — выпадашка пуста, либо ни один пункт не содержит введённой подстроки.
