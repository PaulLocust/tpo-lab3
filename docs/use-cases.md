# Прецеденты использования — T-Путешествия (tbank.ru/travel)

---

| **Прецедент: Просмотр главной страницы T-Путешествий** |
|---|
| **ID:** 1 |
| **Краткое описание:** Открыть главный раздел путешествий и убедиться, что доступны основные подразделы |
| **Главный актер:** Гость |
| **Второстепенные актеры:** нет |
| **Предусловия:** нет |

**Основной поток:**
1. Пользователь переходит на `https://www.tbank.ru/travel/`.
2. Система отображает главную страницу с логотипом T-Банка.
3. Система отображает навигацию по подразделам: Авиа, Отели, Поезда, Туры.
4. Пользователь визуально оценивает доступный функционал.

---

| **Прецедент: Переход между разделами сайта** |
|---|
| **ID:** 2 |
| **Краткое описание:** Перейти из главного раздела в нужный подраздел (авиабилеты, отели, ЖД-билеты, туры) |
| **Главный актер:** Гость |
| **Второстепенные актеры:** нет |
| **Предусловия:** Пользователь находится на любой странице раздела `/travel/` |

**Основной поток:**
1. Пользователь нажимает в шапке на ссылку нужного подраздела.
2. Система выполняет переход на соответствующий URL (`/travel/flights/`, `/travel/hotels/new/`, `/travel/trains/`, `/travel/tours/`).
3. Система отображает форму поиска выбранного подраздела.

---

| **Прецедент: Поиск авиабилетов** |
|---|
| **ID:** 3 |
| **Краткое описание:** Найти подходящие рейсы по маршруту и дате |
| **Главный актер:** Гость |
| **Второстепенные актеры:** нет |
| **Предусловия:** Открыта страница `/travel/flights/` |

**Основной поток:**
1. Пользователь вводит город отправления в поле «Откуда».
2. Система отображает выпадающий список подсказок городов.
3. Пользователь выбирает город из подсказок.
4. Пользователь аналогично заполняет поле «Куда».
5. При необходимости пользователь выбирает дату вылета («extend» UC-08).
6. При необходимости пользователь меняет количество пассажиров и класс («extend» UC-09).
7. Пользователь нажимает кнопку «Найти билеты».
8. Система перенаправляет на страницу выдачи рейсов.

**Альтернативный поток (пустая форма):**

7а. Пользователь не заполнил обязательные поля, система блокирует переход и не открывает выдачу.

---

| **Прецедент: Поиск отелей** |
|---|
| **ID:** 4 |
| **Краткое описание:** Найти отель в указанном городе на нужные даты |
| **Главный актер:** Гость |
| **Второстепенные актеры:** нет |
| **Предусловия:** Открыта страница `/travel/hotels/new/` |

**Основной поток:**
1. Пользователь вводит название города в поле направления.
2. Система отображает подсказки городов («include» UC-10).
3. Пользователь выбирает город из подсказок.
4. При необходимости пользователь задаёт даты заезда/выезда и количество гостей.
5. Пользователь нажимает кнопку «Найти».
6. Система отображает страницу результатов с доступными отелями.

---

| **Прецедент: Поиск ЖД-билетов** |
|---|
| **ID:** 5 |
| **Краткое описание:** Найти ЖД-билеты по маршруту на нужную дату |
| **Главный актер:** Гость |
| **Второстепенные актеры:** нет |
| **Предусловия:** Открыта страница `/travel/trains/` |

**Основной поток:**
1. Пользователь вводит город отправления в поле «Откуда» («include» UC-10).
2. Пользователь вводит город назначения в поле «Куда».
3. При необходимости пользователь выбирает дату поездки.
4. Пользователь нажимает кнопку «Найти».
5. Система отображает страницу с найденными поездами.

---

| **Прецедент: Просмотр раздела «Туры»** |
|---|
| **ID:** 6 |
| **Краткое описание:** Перейти в раздел туров и увидеть форму подбора |
| **Главный актер:** Гость |
| **Второстепенные актеры:** нет |
| **Предусловия:** нет |

**Основной поток:**
1. Пользователь переходит на `/travel/tours/`.
2. Система отображает форму подбора тура с полем направления.

---

| **Прецедент: Обмен городов местами (swap)** |
|---|
| **ID:** 7 |
| **Краткое описание:** Быстро поменять местами «Откуда» и «Куда» в форме поиска авиабилетов |
| **Главный актер:** Гость |
| **Второстепенные актеры:** нет |
| **Предусловия:** Открыта форма авиабилетов («extend» UC-03), оба поля заполнены |

**Основной поток:**
1. Пользователь заполнил поля «Откуда» и «Куда».
2. Пользователь нажимает кнопку обмена городов (значок ⇆).
3. Система обменивает значения полей местами.

---

| **Прецедент: Выбор даты вылета через календарь** |
|---|
| **ID:** 8 |
| **Краткое описание:** Указать дату вылета с помощью виджета календаря |
| **Главный актер:** Гость |
| **Второстепенные актеры:** нет |
| **Предусловия:** Открыта форма авиабилетов («extend» UC-03) |

**Основной поток:**
1. Пользователь нажимает на поле «Туда».
2. Система открывает виджет календаря.
3. Пользователь выбирает нужную дату в календаре.

---

| **Прецедент: Изменение количества пассажиров и класса обслуживания** |
|---|
| **ID:** 9 |
| **Краткое описание:** Указать пассажиров и класс для авиабилета |
| **Главный актер:** Гость |
| **Второстепенные актеры:** нет |
| **Предусловия:** Открыта форма авиабилетов («extend» UC-03) |

**Основной поток:**
1. Пользователь нажимает на селектор пассажиров/класса.
2. Система открывает выпадающую панель с настройками.
3. Пользователь меняет число пассажиров или класс (например, переключает на «Бизнес»).
4. Система отражает изменения в селекторе и применяет их к поиску.

---

| **Прецедент: Автодополнение городов на любой форме** |
|---|
| **ID:** 10 |
| **Краткое описание:** Получать подсказки городов при вводе на любой форме поиска (авиа/отели/ЖД) |
| **Главный актер:** Гость |
| **Второстепенные актеры:** нет |
| **Предусловия:** Открыта одна из форм поиска (`include` UC-03 / UC-04 / UC-05) |

**Основной поток:**
1. Пользователь начинает ввод названия города в поле формы.
2. Система отправляет запрос на сервис подсказок.
3. Система отображает выпадающий список с подсказками.
4. Пользователь выбирает один из вариантов кликом или клавиатурой.
5. Поле заполняется выбранным значением.
